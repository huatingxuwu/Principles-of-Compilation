
============================================================
  测试用例: 嵌套星 (ab)*
============================================================

############################################################
#  正则表达式:  "(a|b)b*"
############################################################

────────────────────────────────────────────────────────────
  第一步: 解析正则表达式 → 抽象语法树 (AST)
────────────────────────────────────────────────────────────
  原始模式: "(a|b)b*"
  AST 结构: 连接(
  并(
    字符 'a',
    字符 'b'
  ),
  星号(
    字符 'b'
  )
)

────────────────────────────────────────────────────────────
  第二步: Thompson 构造法 —— AST → NFA
────────────────────────────────────────────────────────────
  Union: 分别构造左右分支 NFA...
  字符 'a': 新建状态 1 --a--> 2, 起始=1, 接受={2}
  字符 'b': 新建状态 3 --b--> 4, 起始=3, 接受={4}
  Union 合并: 左分支（起始=1, 接受={2}）, 右分支（起始=3, 接受={4}）
  Union 完成: 新建起始 5, 新建接受 {6}, 通过 ε 分叉到左右分支
  连接: 先构造左部 NFA（起始=5, 接受={6}）
  字符 'b': 新建状态 7 --b--> 8, 起始=7, 接受={8}
  Kleene 星: 对子 NFA（起始=7, 接受={8}）加环
  Kleene 星完成: 新建起始 9, 新建接受 {10}, ε-回边从接受态回到子 NFA 起始 7
  连接第2部分: 将左部接受态 {6} 通过 ε 连到右部起始态 9
  连接完成: 起始=5, 接受={10}, 共 10 个状态

  ◆ 构造结果 — NFA 连接关系:
    状态总数: 10
    起始态: 5
    接受态: [10]
    字母表: ['a', 'b']
    状态间连接:
      状态1 经过 'a' 到达 状态2
      状态2 经过 ε 到达 状态6
      状态3 经过 'b' 到达 状态4
      状态4 经过 ε 到达 状态6
      状态5 (起始) 经过 ε 到达 状态1
      状态5 (起始) 经过 ε 到达 状态3
      状态6 经过 ε 到达 状态9
      状态7 经过 'b' 到达 状态8
      状态8 经过 ε 到达 状态7
      状态8 经过 ε 到达 状态10 (接受)
      状态9 经过 ε 到达 状态7
      状态9 经过 ε 到达 状态10 (接受)

────────────────────────────────────────────────────────────
  第三步: 子集构造法 —— NFA → DFA
────────────────────────────────────────────────────────────
  字母表: ['a', 'b']

  初始: ε-闭包({5}) = [1, 3, 5] → DFA 状态 0

  —— 第 1 轮: 处理 DFA 状态 0 (NFA 子集 [1, 3, 5]) [非接受态] ——
    move([1, 3, 5], 'a') = [2]
    ε-闭包([2]) = [2, 6, 7, 9, 10]
    → 新子集, 分配 DFA 状态 1
    DFA 迁移: 状态 0 --a--> 状态 1
    move([1, 3, 5], 'b') = [4]
    ε-闭包([4]) = [4, 6, 7, 9, 10]
    → 新子集, 分配 DFA 状态 2
    DFA 迁移: 状态 0 --b--> 状态 2

  —— 第 2 轮: 处理 DFA 状态 1 (NFA 子集 [2, 6, 7, 9, 10]) [接受态] ——
    move([2, 6, 7, 9, 10], 'a') = {} — 无迁移，跳过
    move([2, 6, 7, 9, 10], 'b') = [8]
    ε-闭包([8]) = [7, 8, 10]
    → 新子集, 分配 DFA 状态 3
    DFA 迁移: 状态 1 --b--> 状态 3

  —— 第 3 轮: 处理 DFA 状态 2 (NFA 子集 [4, 6, 7, 9, 10]) [接受态] ——
    move([4, 6, 7, 9, 10], 'a') = {} — 无迁移，跳过
    move([4, 6, 7, 9, 10], 'b') = [8]
    ε-闭包([8]) = [7, 8, 10]
    → 已有子集, 对应 DFA 状态 3
    DFA 迁移: 状态 2 --b--> 状态 3

  —— 第 4 轮: 处理 DFA 状态 3 (NFA 子集 [7, 8, 10]) [接受态] ——
    move([7, 8, 10], 'a') = {} — 无迁移，跳过
    move([7, 8, 10], 'b') = [8]
    ε-闭包([8]) = [7, 8, 10]
    → 已有子集, 对应 DFA 状态 3
    DFA 迁移: 状态 3 --b--> 状态 3

  子集构造完成: 共 4 个 DFA 状态
  DFA 状态 → NFA 子集 对应关系:
    状态 0 ↔ NFA 子集 [1, 3, 5]
    状态 1 ↔ NFA 子集 [2, 6, 7, 9, 10] *接受
    状态 2 ↔ NFA 子集 [4, 6, 7, 9, 10] *接受
    状态 3 ↔ NFA 子集 [7, 8, 10] *接受

  ◆ 构造结果 — DFA 连接关系:
    [子集构造 DFA] 状态总数: 4
    起始态: 0
    接受态: [1, 2, 3]
    字母表: ['a', 'b']
    状态间连接:
      状态0 (起始) 经过 'a' 到达 状态1 (接受)
      状态0 (起始) 经过 'b' 到达 状态2 (接受)
      状态1 经过 'b' 到达 状态3 (接受)
      状态2 经过 'b' 到达 状态3 (接受)
      状态3 经过 'b' 到达 状态3 (接受)

────────────────────────────────────────────────────────────
  第四步: Hopcroft 算法 —— DFA 最小化
────────────────────────────────────────────────────────────

  初始划分:
    非接受态块: [0]
    接受态块:   [1, 2, 3]

  —— 第 1 轮细化: 取块 A = [0] ——
    本轮无分裂，划分稳定。

  —— 第 2 轮细化: 取块 A = [1, 2, 3] ——
    本轮无分裂，划分稳定。

  最终等价类划分:
    块 0: [0]
    块 1: [1, 2, 3]

  重新编号后的映射: 旧 DFA 状态 → 最小化 DFA 状态
    DFA 状态 [0] → 新状态 0
    DFA 状态 [1, 2, 3] → 新状态 1 *接受

  ◆ 最终结果 — 最小化 DFA 连接关系:
    [最小化 DFA] 状态总数: 2
    起始态: 0
    接受态: [1]
    字母表: ['a', 'b']
    状态间连接:
      状态0 (起始) 经过 'a' 到达 状态1 (接受)
      状态0 (起始) 经过 'b' 到达 状态1 (接受)
      状态1 经过 'b' 到达 状态1 (接受)
